SECTION 1: PROJECT OVERVIEW
================================================================================

The AHJ Engine is a FastAPI-based web application that manages regulatory and
jurisdiction data for solar installation authorities. The system includes:

✅ Complete PostgreSQL database with 22 interconnected tables
✅ RESTful API endpoints for all business entities
✅ SQLAdmin web interface for data management with enhanced UI
✅ Excel data import system from regulatory data files
✅ Complex data relationships and validation rules
✅ User authentication and role-based access control
✅ Real-time form UI with tag-based multi-select controls

Key Components:
- Database: PostgreSQL (22 tables, complex relationships)
- API Framework: FastAPI with async support
- Web Interface: SQLAdmin with extensive customization
- Data Import: Excel files → Python openpyxl → Database
- Frontend: HTML/CSS/Vanilla JavaScript (no heavy dependencies)

Location: scripts/import/import_labels.py
Purpose: Import master data (labels, categories, equipment) from Excel

Key Components:

1. MasterDataImporter Class
   - Handles Excel file operations
   - Manages database session and transactions
   - Tracks import statistics

2. File Loading
   ✅ Loads "Illumine-i X LabelFriday master data.xlsx"
   ✅ Loads "Label filter data.xlsx"
   ✅ Handles multiple sheets within Excel files

3. Data Processing Pipeline

   a) Read Headers
      - Read first row of Excel sheet
      - Map column names to database field names
      - Skip unmapped columns

   b) Iterate Rows
      - Process each data row in succession
      - Extract values for each column
      - Skip empty rows

   c) Data Validation
      - Validate required fields (name, description, etc.)
      - Skip rows with missing critical data
      - Handle different data types (strings, numbers, dates)

   d) Duplicate Detection
      - Check if record already exists in database
      - Query by primary fields (usually 'name')
      - Skip duplicate entries
      - Report statistics

   e) Data Transformation
      - Clean and format string values
      - Convert types as needed
      - Prepare for database insertion

   f) Object Creation
      - Create ORM objects (Label, Category, Equipment instances)
      - Set relationships (e.g., equipment → categories)
      - Store in session for batch insert

   g) Transaction Commit
      - Batch insert all new records at once
      - Commit transaction if successful
      - Rollback on error with logging

4. Statistics Tracking
   - Counts created: Total new records inserted
   - Counts skipped: Duplicate or invalid records
   - Counts failed: Records that couldn't be processed
   - Errors logged: Details of any processing failures

3.5 PROCESSING FLOWCHART - DETAILED STEPS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

STEP-BY-STEP DATA FLOW:

Step 1: Initialize
   └─→ Get database session
   └─→ Load Excel file using openpyxl
   └─→ Identify sheets to import

Step 2: Create ORM Session
   └─→ Create SQLAlchemy session
   └─→ Establish database connection
   └─→ Set echo=False for silent operations

Step 3: For Each Sheet in Excel File
   └─→ Read sheet name
   └─→ Determine sheet type (labels, categories, equipment, etc.)
   └─→ Get sheet data

Step 4: Parse Headers
   └─→ Read row 1 from sheet
   └─→ Create mapping: Excel_Column → DataBase_Field
   └─→ Example:
       Excel Column "Label Name" → Database Field "name"
       Excel Column "Description" → Database Field "description"
       Excel Column "Color Code" → Database Field "color"

Step 5: Process Data Rows
   For each row in sheet (starting from row 2):
   
   a) Extract Values
      ├─→ Read all column values
      ├─→ Convert Excel data types
      └─→ Handle None/empty values
   
   b) Validate
      ├─→ Check required fields present
      ├─→ Skip rows with critical missing data
      ├─→ Validate data formats
      └─→ Handle parsing errors
   
   c) Check Duplicates
      ├─→ Query: SELECT * WHERE name = row_name
      ├─→ Found? → Skip this row, increment skip_count
      ├─→ Not found? → Continue to create
   
   d) Create ORM Object
      ├─→ label = Label(name=row.name, description=row.desc, color=row.color)
      ├─→ Set all properties from Excel row
      ├─→ Add to session.add(label)
   
   e) Handle Relationships
      ├─→ If equipment needs categories:
      ├─────→ Find category by name
      ├─────→ association = CombinationMapper(equipment_id, category_id)
      ├─────→ session.add(association)
      └─→ Add relationships through junction tables

Step 6: Batch Commit
   └─→ After processing all rows in sheet:
   └─→ Try:
       ├─→ Call session.commit()
       ├─→ Flush changes to database
       └─→ Return success count
   └─→ Except:
       ├─→ Log error details
       ├─→ session.rollback()
       ├─→ Report failure

Step 7: Report Statistics
   └─→ Print summary:
       ├─→ Total records found in Excel
       ├─→ Records created (inserted)
       ├─→ Records skipped (duplicates)
       ├─→ Records failed (errors)
       └─→ Overall success/failure status

SECTION 8: TROUBLESHOOTING & COMMON ISSUES

8.1 IMPORT ISSUES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Issue: "FileNotFoundError: Excel file not found"
Solution:
  1. Check file exists in app/data/master/ directory
  2. Verify file name matches exactly in script
  3. Check file permissions are readable
  4. Ensure full path in os.path.join() call

Issue: "Foreign key constraint violation"
Solution:
  1. Import parent tables first (countries before states)
  2. Verify parent records exist before importing children
  3. Check foreign key references in import script
  4. Run validate_setup.py to check relationships

Issue: "Duplicate key violation"
Solution:
  1. Check duplicate detection query in script
  2. Verify unique constraints on columns
  3. Review UNIQUE clauses in model definitions
  4. Consider if duplicate detection should be skipped

Issue: "No module named 'openpyxl'"
Solution:
  1. pip install openpyxl
  2. Check requirements.txt includes it
  3. Verify virtual environment is activated

8.2 WEB INTERFACE ISSUES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Issue: Tag input not appearing in form
Solution:
  1. Check JavaScript console for errors (F12)
  2. Verify tag_input.js loaded (check Network tab)
  3. Check sqladmin_custom.css loaded
  4. Ensure DOMContentLoaded event fires
  5. Check browser supports data-* attributes

Issue: Tags not syncing to form submission
Solution:
  1. Inspect hidden select element in browser
  2. Check if updateSelectElement() called before form submit
  3. Verify form has proper hidden select[multiple]
  4. Check server-side receives form data correctly

Issue: Color picker not visible or too large
Solution:
  1. Check sqladmin_custom.css loaded
  2. Verify CSS rule `input[type="color"] { width: 80px; }`
  3. Check for CSS conflicts from other stylesheets
  4. Clear browser cache and refresh

Issue: 500 Internal Server Error
Solution:
  1. Check application logs for error details
  2. Verify database connection working
  3. Check form_overrides not causing issues
  4. Ensure all model relationships defined correctly
  5. Check WTForms field compatibility

8.3 DATABASE ISSUES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
