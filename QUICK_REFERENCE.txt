## Error You Showed (Explained)

```
UniqueViolation: duplicate key "states_abbrev_key"
Key (abbrev)=(TX) already exists
```

**What happened:** You tried to add a test state "Aadil S" with abbreviation "TX", but TX already exists for Texas. This is **correct behavior** - the database prevented duplicate abbreviations.

**Why it matters:** Each state gets ONE official abbreviation. The unique constraint is working perfectly.

**Result:** No action needed - the system did its job! ✓

## To Deploy to Render (5 Steps)

### Step 1: Push Code
```bash
git add .
git commit -m "AHJ Engine ready"
git push origin main
```

### Step 2: Go to Render
Visit: https://render.com/dashboard

### Step 3: Create Web Service
```
Click: New → Web Service
Select: Your GitHub repo
Branch: main
```

### Step 4: Configure
```
Build:   bash build.sh
Start:   gunicorn -w 4 -k uvicorn.workers.UvicornWorker app.main:app

Environment Variables:
  DATABASE_URL = [from Render PostgreSQL]
  DEBUG = false
  SECRET_KEY = [generate: python -c "import secrets; print(secrets.token_urlsafe(32))"]
```

### Step 5: Deploy
```
Click: Create Web Service
Wait: 5-10 minutes
Done! ✓
```

**Access:**
- Admin: `https://your-app.onrender.com/admin`
- API: `https://your-app.onrender.com/api/v1/label/`

## How to Use Locally

### 1. Start Server
```bash
python -m uvicorn app.main:app --reload
```

### 2. Access Admin
```
http://localhost:8000/admin
```

### 3. Create Code
```
Admin → Codes → New Code
Title: 2021 National Electrical Code
Type: NEC
Category: Electrical
Amendment: STANDARD
Save ✓
```

### 4. Create Label
```
Admin → Labels → New Label
Select UPC: LF-001
Auto-fills:
  - Length: 4
  - Width: 1
  - Label Name: Write in DC label
  - Description: [auto-filled]
  - Colors: Orange/Black
Save ✓
```

### 5. Test API
```bash
# All labels
curl http://localhost:8000/api/v1/label/

# Specific label
curl http://localhost:8000/api/v1/label/1/details

# By UPC
curl http://localhost:8000/api/v1/label/by-upc/LF-001/details
```
