================================================================================
ARCHITECTURE OVERVIEW
================================================================================

Your project follows this architecture:

DATABASE → MODELS → SCHEMAS → SERVICES → API ROUTES → ADMIN PANEL

1. DATABASE: PostgreSQL tables (defined in .env with DATABASE_URL)
2. MODELS: SQLAlchemy ORM models (app/models/)
3. SCHEMAS: Pydantic validation schemas (app/schemas/)
4. SERVICES: Business logic layer (app/services/)
5. API ROUTES: FastAPI endpoints (app/api/v1/)
6. ADMIN PANEL: SQLAdmin views (app/admin/admin.py)

================================================================================
STEP-BY-STEP GUIDE: ADD A NEW ENTITY (EXAMPLE: "Department")
================================================================================

SCENARIO: You want to add a Department entity to manage different departments
with name, code, and description fields.

---

STEP 1: CREATE THE MODEL
========================

File: app/models/department.py

```python
from sqlalchemy import Column, Integer, String, Text
from app.core.database import Base

class Department(Base):
    __tablename__ = "departments"

    id = Column(Integer, primary_key=True, index=True)
    name = Column(String(100), unique=True, nullable=False)
    code = Column(String(50), unique=True, nullable=False)
    description = Column(Text, nullable=True)
```

KEY POINTS:
- File name should match entity name (lowercase): department.py
- Class name should be PascalCase: Department
- __tablename__ should be plural: departments
- Always include: id (primary key), and required fields
- Use nullable=False for required fields
- Use unique=True for fields that should be one-of-a-kind
- Use index=True for fields you'll filter by frequently

STEP 2: CREATE THE SCHEMA
==========================

File: app/schemas/department.py

```python
from pydantic import BaseModel
from app.schemas import BaseSchema

class DepartmentBase(BaseSchema):
    """Base department schema with common fields"""
    name: str
    code: str
    description: str | None = None

class DepartmentCreate(DepartmentBase):
    """Schema for creating a new department"""
    pass

class DepartmentUpdate(BaseSchema):
    """Schema for updating a department (all fields optional)"""
    name: str | None = None
    code: str | None = None
    description: str | None = None

class DepartmentResponse(DepartmentBase):
    """Schema for API response"""
    id: int
    
    class Config:
        from_attributes = True  # ✅ Use this! (Not orm_mode)

class DepartmentList(BaseSchema):
    """Schema for list responses"""
    id: int
    name: str
    code: str
```

KEY POINTS:
- Create separate schemas for different operations:
  * DepartmentBase: Common fields
  * DepartmentCreate: For POST requests (no id)
  * DepartmentUpdate: For PUT/PATCH requests (optional fields)
  * DepartmentResponse: For responses (includes id)
  * DepartmentList: For list responses (simplified)
- Always inherit from BaseSchema (for Pydantic V2 config)
- Use Config with from_attributes = True (required for Pydantic V2)
- Use str | None for optional fields
- Always use from_attributes in response schemas

STEP 3: CREATE THE SERVICE
============================

File: app/services/department_service.py

```python
from sqlalchemy.orm import Session
from app.models.department import Department
from app.schemas.department import DepartmentCreate, DepartmentUpdate
from fastapi import HTTPException

class DepartmentService:
    
    def get_all(self, db: Session):
        """Get all departments"""
        return db.query(Department).all()
    
    def get_by_id(self, db: Session, department_id: int):
        """Get department by ID"""
        department = db.query(Department).filter(
            Department.id == department_id
        ).first()
        
        if not department:
            raise HTTPException(status_code=404, detail="Department not found")
        
        return department
    
    def get_by_code(self, db: Session, code: str):
        """Get department by code"""
        return db.query(Department).filter(Department.code == code).first()
    
    def create(self, db: Session, department: DepartmentCreate):
        """Create new department"""
        # Check if department with this code already exists
        existing = self.get_by_code(db, department.code)
        if existing:
            raise HTTPException(
                status_code=400, 
                detail=f"Department with code '{department.code}' already exists"
            )
        
        db_department = Department(**department.dict())
        db.add(db_department)
        db.commit()
        db.refresh(db_department)
        
        return db_department
    
    def update(self, db: Session, department_id: int, department: DepartmentUpdate):
        """Update department"""
        db_department = self.get_by_id(db, department_id)
        
        # Update only provided fields
        update_data = department.dict(exclude_unset=True)
        for field, value in update_data.items():
            setattr(db_department, field, value)
        
        db.commit()
        db.refresh(db_department)
        
        return db_department
    
    def delete(self, db: Session, department_id: int):
        """Delete department"""
        db_department = self.get_by_id(db, department_id)
        db.delete(db_department)
        db.commit()
        
        return {"message": "Department deleted successfully"}
```

KEY POINTS:
- Service contains business logic
- Always validate before operations
- Use HTTPException for errors (status_code, detail)
- Use db.add(), db.commit(), db.refresh() for database operations
- Use dict(exclude_unset=True) for partial updates
- Return meaningful responses
- Handle edge cases (duplicates, not found, etc.)

STEP 4: CREATE API ROUTES
==========================

File: app/api/v1/department_api.py

```python
from fastapi import APIRouter, Depends, HTTPException
from sqlalchemy.orm import Session
from app.core.database import get_db
from app.schemas.department import (
    DepartmentCreate, DepartmentUpdate, DepartmentResponse, DepartmentList
)
from app.services.department_service import DepartmentService

class DepartmentAPI:
    def __init__(self):
        self.router = APIRouter()
        self.service = DepartmentService()
        self._setup_routes()
    
    def _setup_routes(self):
        @self.router.get("/", response_model=list[DepartmentList])
        def list_departments(db: Session = Depends(get_db)):
            """Get all departments"""
            return self.service.get_all(db)
        
        @self.router.get("/{department_id}", response_model=DepartmentResponse)
        def get_department(department_id: int, db: Session = Depends(get_db)):
            """Get department by ID"""
            return self.service.get_by_id(db, department_id)
        
        @self.router.post("/", response_model=DepartmentResponse)
        def create_department(
            department: DepartmentCreate, 
            db: Session = Depends(get_db)
        ):
            """Create new department"""
            return self.service.create(db, department)
        
        @self.router.put("/{department_id}", response_model=DepartmentResponse)
        def update_department(
            department_id: int, 
            department: DepartmentUpdate, 
            db: Session = Depends(get_db)
        ):
            """Update department"""
            return self.service.update(db, department_id, department)
        
        @self.router.delete("/{department_id}")
        def delete_department(
            department_id: int, 
            db: Session = Depends(get_db)
        ):
            """Delete department"""
            return self.service.delete(db, department_id)
```

KEY POINTS:
- Create a class to hold all routes
- Initialize router and service in __init__
- Use _setup_routes() to register all endpoints
- Use response_model for automatic validation
- Use Depends(get_db) to inject database session
- Use appropriate HTTP methods: GET, POST, PUT, DELETE
- Include docstrings for Swagger documentation
- Use URL parameters for IDs: /{id}
- Use request body for data

ENDPOINTS CREATED:
- GET /api/v1/department/ - List all
- GET /api/v1/department/{id} - Get by ID
- POST /api/v1/department/ - Create
- PUT /api/v1/department/{id} - Update
- DELETE /api/v1/department/{id} - Delete

STEP 10: HANDLE RELATIONSHIPS (IF NEEDED)
==========================================

Example: If Department has many Employees, use relationships:

FILE: app/models/department.py
```python
from sqlalchemy import Column, Integer, String, Text
from sqlalchemy.orm import relationship
from app.core.database import Base

class Department(Base):
    __tablename__ = "departments"

    id = Column(Integer, primary_key=True, index=True)
    name = Column(String(100), unique=True, nullable=False)
    code = Column(String(50), unique=True, nullable=False)
    description = Column(Text, nullable=True)
    
    # Relationship
    employees = relationship("Employee", backref="department")
```

FILE: app/models/employee.py
```python
from sqlalchemy import Column, Integer, String, ForeignKey
from sqlalchemy.orm import relationship
from app.core.database import Base

class Employee(Base):
    __tablename__ = "employees"

    id = Column(Integer, primary_key=True, index=True)
    name = Column(String(100), nullable=False)
    department_id = Column(Integer, ForeignKey("departments.id"))
    
    # Relationship
    department = relationship("Department", backref="employees")
```

KEY POINTS:
- Use ForeignKey("table_name.column_name")
- Use relationship() for ORM relationships
- Use backref for reverse relationships
- Filter in services: db.query(Department).filter(Department.id == id).first()

In service:
```python
def get_paginated(self, db: Session, skip: int = 0, limit: int = 10):
```

PATTERN 3: SORTING
==================

In service:
```python
def get_sorted(self, db: Session, sort_by: str = "name", sort_order: str = "asc"):
    query = db.query(Department)
    
    if sort_by == "name":
        column = Department.name
    elif sort_by == "code":
        column = Department.code
    else:
        column = Department.id
    
    if sort_order.lower() == "desc":
        return query.order_by(column.desc()).all()
    else:
        return query.order_by(column.asc()).all()
```

In API:
```python
@self.router.get("/")
def list_departments(
    sort_by: str = "name",
    sort_order: str = "asc",
    db: Session = Depends(get_db)
):
    return self.service.get_sorted(db, sort_by, sort_order)
```

PATTERN 4: BULK OPERATIONS
===========================

In service:
```python
def bulk_create(self, db: Session, departments: list[DepartmentCreate]):
    """Create multiple departments at once"""
    db_departments = [Department(**dept.dict()) for dept in departments]
    db.add_all(db_departments)
    db.commit()
    
    for dept in db_departments:
        db.refresh(dept)
    
    return db_departments

def bulk_delete(self, db: Session, ids: list[int]):
    """Delete multiple departments"""
    db.query(Department).filter(Department.id.in_(ids)).delete()
    db.commit()
    
    return {"message": f"Deleted {len(ids)} departments"}
```

PATTERN 5: VALIDATION
======================

In schema:
```python
from pydantic import BaseModel, field_validator, Field

class DepartmentCreate(BaseSchema):
    name: str = Field(..., min_length=1, max_length=100)
    code: str = Field(..., min_length=1, max_length=50)
    description: str | None = None
    
    @field_validator('code')
    def validate_code(cls, v):
        if not v.isalnum():
            raise ValueError('Code must be alphanumeric')
        return v.upper()
```

PATTERN 6: SOFT DELETE (Don't permanently delete)
==================================================

In model:
```python
from sqlalchemy import Column, Integer, String, DateTime
from datetime import datetime

class Department(Base):
    __tablename__ = "departments"

    id = Column(Integer, primary_key=True, index=True)
    name = Column(String(100), nullable=False)
    code = Column(String(50), nullable=False)
    deleted_at = Column(DateTime, nullable=True)  # Soft delete marker
```

In service:
```python
def get_all(self, db: Session):
    """Get only active departments"""
    return db.query(Department).filter(Department.deleted_at == None).all()

def delete(self, db: Session, department_id: int):
    """Soft delete - mark as deleted instead of removing"""
    db_department = self.get_by_id(db, department_id)
    db_department.deleted_at = datetime.now()
    db.commit()
    return {"message": "Department deleted (soft delete)"}
```

================================================================================
TROUBLESHOOTING COMMON ISSUES
================================================================================

ISSUE 1: "ModuleNotFoundError: No module named 'app.models.department'"
SOLUTION: Make sure the file path is exactly: app/models/department.py

ISSUE 2: "Table 'departments' doesn't exist"
SOLUTION: Run: python init_db.py
(or: venv\Scripts\alembic revision --autogenerate -m "Add Department"
     venv\Scripts\alembic upgrade head)

ISSUE 3: Pydantic validation error
SOLUTION: Check schema uses from_attributes = True (not orm_mode)

ISSUE 4: HTTPException not returning proper error
SOLUTION: Make sure service raises HTTPException, not returning None

ISSUE 5: Admin panel shows but can't list items
SOLUTION: Verify the admin class column_list matches actual model fields

ISSUE 6: API returns 500 error
SOLUTION: Check server logs for the exact error message

ISSUE 7: Schema validation fails
SOLUTION: Verify field types match: str, int, bool, list, etc.
         Check optional fields use: field: str | None = None
